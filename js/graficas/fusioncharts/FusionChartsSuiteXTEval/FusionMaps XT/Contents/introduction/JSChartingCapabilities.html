Introducing JavaScript maps

Starting FusionMaps XT, you can now render maps on devices where Flash player is not supported (like in iPhone/iPad), is not installed or is disabled. For this, FusionMaps XT uses the built-in JavaScript rendering capabilities. The maps, rendered by this module, purely use JavaScript and do not need Flash Player at all.

FusionMaps XT gives you the option of either using automatic fallback, wherein the maps figure out the best mode of rendering (either Flash or JavaScript, based on availability of Flash Player), or lets you configure whether to render maps exclusively using JavaScript.

If you are already using FusionMaps XT and wish to add JavaScript fallback, you just need to replace the old SWF files with the new ones and copy all the JavaScript files from the Maps folder of the Download Pack of FusionMaps XT - without changing a single line of code.

In case you wish to render JavaScript maps only, irrespective of whether Flash Player is installed or not, all you need to do is add a single line of code to your existing implementation. This has been covered in forthcoming sections. Meanwhile, let's get a small overview of FusionMaps XT JavaScript capabilities.

How it works?

FusionMaps XT features a completely new JavaScript class - FusionCharts JavaScript Class. It caters to all the products of FusionCharts Suite XT that includes FusionMaps XT. FusionCharts JavaScript Class (in FusionMaps XT pack) consists of the following :

  • Core Libraries: FusionCharts.js, FusionCharts.HC.js, FusionCharts.HC.Maps.js and jquery.min.js. These four files are present in Download Pack > Maps folder.
  • Map files: Each map is defined in a single JavaScript file, For example, the map file for World is FusionCharts.HC.world.js. All the JavaScript map files are present in Download Pack > Maps folder.

You will only need to include FusionCharts.js in your web page (do not forget to copy the rest of the files to your web application). The rest of the JavaScript files will be automatically loaded on-demand by FusionCharts.js.

FusionCharts JavaScript Class determines whether Flash player is not available in a device. It automatically switches the rendering mode to JavaScript when Flash Player is unavailable. You can also force FusionCharts JavaScript Class to always render JavaScript maps.
